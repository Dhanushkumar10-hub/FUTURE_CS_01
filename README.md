# Cyber Security Internship – Task 1

## Task Title
Vulnerability Assessment Report for a Public Website (Read-Only)

## Internship Program
Future Interns – Cyber Security (2026)

## Tested Website
testphp.vulnweb.com  
(This is a publicly available demo website designed for security testing and educational purposes.)

## Objective
To perform a read-only vulnerability assessment on a public website, identify common security misconfigurations, classify risks, and provide business-friendly remediation recommendations.

## Scope
- Public-facing pages only
- Passive security analysis
- HTTP header and configuration review

## Out of Scope
- Exploitation
- Login or authentication testing
- Brute-force attacks
- Denial-of-Service (DoS) attacks

## Tools Used
- Nmap
- Browser Developer Tools
- Mozilla HTTP Observatory
- Canva

## Key Findings
- Missing Content Security Policy (CSP)
- Missing X-Frame-Options Header
- Missing X-Content-Type-Options Header
- No HTTPS Enforcement / Missing HSTS
- Server Information Disclosure

## Repository Structure
- Vulnerability_Assessment_Report_Task1.pdf – Final assessment report
- evidence/ – Supporting screenshots for findings

## Disclaimer
This assessment was conducted strictly for educational purposes as part of the Future Interns Cyber Security Internship Program.  
All testing was passive, ethical, and performed on a publicly available demo website. No systems were harmed or exploited.
